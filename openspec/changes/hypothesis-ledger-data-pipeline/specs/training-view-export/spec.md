## ADDED Requirements

### Requirement: Prefix-only, oracle-free input
A sample's input SHALL contain only content up to the turning-point `cut_step` plus current feedback, and MUST NOT contain gold, oracle answer, hidden-test assertions, or `file:line`. Gold is the oracle-step midwife only and never enters an input.

#### Scenario: Future-leaking or oracle-bearing input rejected
- **WHEN** an input contains post-cut content or any gold / oracle / `file:line` content
- **THEN** the exporter drops the sample

### Requirement: Minimal fixed-section target
The assistant target SHALL be the fixed-section block `STATUS / WHY / RISK / MISSING EVIDENCE / NEXT MOVE / AVOID` — no nested JSON, candidate lists, or metadata. Provenance (`generated_from_full_audit`, `cut_step`, `leakage_level`) lives in an out-of-band sidecar.

#### Scenario: Target is sections, not a schema
- **WHEN** a target is emitted
- **THEN** it contains only the six sections, with no nested lists or metadata in the prompt or target

### Requirement: Pre-submit vs post-submit discipline
A pre-submit sample's input MUST NOT carry a hidden PASS_TO_PASS result, and its target SHALL cap at `weak_support` + band-aid risk + "probe before submit"; the pre-submit target SHALL be regenerated by a separate prefix-only strong-model call (blind to outcome). A post-submit sample MAY include harness feedback and MAY state that the failure refutes the boundary.

#### Scenario: Pre-submit target does not predict hidden failure
- **WHEN** a pre-submit sample comes from a run that later failed PASS_TO_PASS
- **THEN** the target says support is weak / band-aid risk high / probe more, and does not assert that the hidden test will fail

### Requirement: Quality gate and instance-id split
A positive sample SHALL pass the gate (old hypothesis explicit, evidence explicit, turning point explicit, target direction-level, no gold / hidden assertion, `leakage_level` in {none, direction_only}) or be dropped / rerouted with a logged reason. Splits SHALL be partitioned by `instance_id` with empty pairwise intersection.

#### Scenario: Failing sample rerouted, not trained
- **WHEN** a candidate fails any gate condition
- **THEN** it is excluded from positive SFT, its drop reason logged, and it may go to critique / eval instead

#### Scenario: Splits disjoint
- **WHEN** `dataset/splits/{train,dev,test}.json` are built
- **THEN** no `instance_id` appears in more than one split

### Requirement: Held-out, gold-anchored, independent grader
The held-out turning-point eval SHALL use test-split instances, anchored on **gold / protocol turning points** (the audit generator MUST NOT write the eval anchors), scored by a grader whose model version / family differs from the audit labeler, with rubric + scale frozen before scoring. It SHALL report behavioral metrics judgable from a single fixed-section target (`audit_quality`, `support_calibration`, `band_aid_detection`, `redirect_quality`) and SHALL NOT claim SWE-bench resolve improvement.

#### Scenario: Independent anchors and grader
- **WHEN** base and SFT are scored
- **THEN** both use the same frozen grader (a different model than the labeler) and the gold / protocol anchors, and the report makes no SWE-resolve claim

#### Scenario: Eval instances held out
- **WHEN** the eval set is assembled
- **THEN** every eval instance comes from the test split
